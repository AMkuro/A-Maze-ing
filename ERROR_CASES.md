# Error Cases

This document lists the cases where `a_maze_ing.py` reports an error.

When the CLI catches an error during execution, it prints the message to
`stderr` with this prefix and exits with status code `1`:

```text
Configuration error: <message>
```

If the command-line argument count is wrong, it prints usage to `stderr` and
exits with status code `2`:

```text
Usage: python3 a_maze_ing.py <config.txt>
```

## Configuration File Read Errors

These happen before parsing `KEY=VALUE` lines.

| Case | Message |
| --- | --- |
| The config file path does not exist | `Configuration file not found: <path>` |
| The config file exists but cannot be read | `Could not read configuration file: <path>` |

## Configuration Syntax Errors

Blank lines and lines starting with `#` are ignored. Every other line must be
one `KEY=VALUE` pair.

| Case | Example | Message |
| --- | --- | --- |
| A non-comment line has no `=` | `WIDTH 10` | `Invalid config line N: expected KEY=VALUE.` |
| The key is empty | `=10` | `Invalid config line N: key must not be empty.` |
| The value is empty | `WIDTH=` | `Invalid config line N: value must not be empty.` |
| A key appears more than once | two `WIDTH=` lines | `Duplicate config key: WIDTH.` |
| A key is not supported | `ALGORITHM=dfs` | `Unknown config key: ALGORITHM.` |

Supported keys are:

- `WIDTH`
- `HEIGHT`
- `ENTRY`
- `EXIT`
- `OUTPUT_FILE`
- `PERFECT`
- `SEED`

`SEED` is optional. All other keys are mandatory.

## Missing Required Keys

If a mandatory key is missing, the message is:

```text
Missing required config key: <KEY>.
```

This applies to:

- `WIDTH`
- `HEIGHT`
- `ENTRY`
- `EXIT`
- `OUTPUT_FILE`
- `PERFECT`

## Configuration Value Errors

These errors come from validation after the file has been parsed.

| Key | Error case | Message shape |
| --- | --- | --- |
| `WIDTH` | Not an integer | `WIDTH: Input should be a valid integer...` |
| `WIDTH` | `0` or negative | `WIDTH: Input should be greater than 0` |
| `HEIGHT` | Not an integer | `HEIGHT: Input should be a valid integer...` |
| `HEIGHT` | `0` or negative | `HEIGHT: Input should be greater than 0` |
| `ENTRY` | Not in `x,y` format | `ENTRY: must be coordinates in x,y format.` |
| `ENTRY` | Contains non-integer coordinates | `ENTRY: must be coordinates in x,y format.` |
| `ENTRY` | Outside the maze bounds | `ENTRY must be inside the maze bounds.` |
| `EXIT` | Not in `x,y` format | `EXIT: must be coordinates in x,y format.` |
| `EXIT` | Contains non-integer coordinates | `EXIT: must be coordinates in x,y format.` |
| `EXIT` | Outside the maze bounds | `EXIT must be inside the maze bounds.` |
| `ENTRY`/`EXIT` | Same cell | `ENTRY and EXIT must be different.` |
| `OUTPUT_FILE` | Empty after trimming | `Invalid config line N: value must not be empty.` |
| `OUTPUT_FILE` | Contains `NUL`, newline, carriage return, `<`, `>`, `|`, `*`, `?`, or `"` | `OUTPUT_FILE: OUTPUT_FILE contains invalid characters: ...` |
| `PERFECT` | Not parseable as a boolean | `PERFECT: Input should be a valid boolean...` |
| `SEED` | Not an integer | `SEED: Input should be a valid integer...` |

## Maze Generation Errors

These happen after the configuration has been loaded. That means the config
file can be syntactically valid and pass value validation, but still fail during
generation because the requested maze cannot be generated by this implementation.

| Case | Message |
| --- | --- |
| `WIDTH=1` or `HEIGHT=1` | `A maze with a single path is not a maze.` |
| The generator cannot find any usable non-`42` cell | `No valid starting cell found for maze generation.` |
| `PERFECT=False` but every removable wall would create a 3x3 open area | `PERFECT = False but every removable wall creates a 3x3 open area.` |

Examples in `config/`:

- `invalid_single_column_maze.txt`
- `invalid_single_row_maze.txt`

The program may also print a warning, not an error, when the visual `42`
pattern cannot be embedded:

| Case | Warning |
| --- | --- |
| The maze is too small for the `42` pattern | `"42" pattern has omitted by the maze size.` |
| `ENTRY` or `EXIT` would overlap the `42` pattern | `"42" pattern has omitted by the place of entry and exit` |

## Maze Solving Errors

These are internal consistency errors after generation.

| Case | Message |
| --- | --- |
| No path exists from `ENTRY` to `EXIT` | `Maze has no solution` |
| The BFS predecessor data is inconsistent | `Invalid path data` |

## Maze Validation Errors

These are internal consistency checks run after generation.

| Case | Message |
| --- | --- |
| Adjacent cells disagree about a shared east/west or north/south wall | `Wall Consistency Error:pos (x,y)` |
| The grid height does not match `HEIGHT` | `Check Bounds Error: grid size exceed or under of the range.` |
| A row width does not match `WIDTH` | `Check Bounds Error: grid size exceed or under of the range.` |
| An external border wall is open | `There is hole at the external borders.` |
| A fully open 3x3 area exists | `There is a 3*3 open area.` |
| `PERFECT=False`, but the generated maze still has no loop | `Maze is still perfect, but should be imperfect.` |

## Output Errors

After generation, the output file is written to `OUTPUT_FILE`. The config can
be valid and generation can succeed, but writing can still fail.

| Case | Message |
| --- | --- |
| The output path cannot be written, for example because its parent directory does not exist or permission is denied | Python `OSError` message, prefixed by `Configuration error:` in the CLI |

Example in `config/`:

- `invalid_output_parent_missing.txt`
